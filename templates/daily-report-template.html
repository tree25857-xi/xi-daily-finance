入門基礎 Week 1–2 · 安裝 & 第一個訊息平臺
目標：把 OpenClaw 真正裝起來、接上 Telegram / Discord 其中一個，並啟用安全且實用的第一批官方 Skills。

1. 系統與帳號準備

  • 一台長期在線的機器：雲端主機 / NAS / 家用機器人主機皆可。
  • 建議安裝 Docker（或 Node.js / Python，視你實際教學路線而定）。
  • 準備至少一組 LLM API 金鑰（OpenAI / Anthropic / 其他相容供應商）。
  • 建立 Telegram / Discord 等你打算優先使用的消息平臺帳號。
推薦：先從 Telegram 開始，錯誤資訊最清楚

2. 安全與風險提醒

  • 一開始只啟用「讀取型」或低風險 Skills（摘要、查詢、筆記寫入等）。
  • 避免立刻開啟「金流 / 下單 / 密碼管理 / 家庭門鎖」等高風險技能。
  • 使用 .env 或 Secrets 管理金鑰與密碼，避免直接寫死在程式碼中。
中風險：有寫入權限的 Skills（Notion / Obsidian / Email） 高風險：金融 / 家電 / 金流相關 Skills

安裝與基本設定流程（示意）：

實際指令請依你的安裝方式修改，此處提供一個通用結構，方便你在 Google Sites 裡分段說明。

步驟一：下載與初始化

以 CLI 方式為例：

# 1. 下載 / 安裝 OpenClaw（依實際文件替換）
curl -L https://your-install-script.sh | bash

# 2. 初始化設定
openclaw init

# 3. 檢查版本與基本狀態
openclaw status

初始化完成後，會在你的設定目錄中產生預設的 config.yml 或等價設定檔。

步驟二：編輯設定檔

  • 填入 LLM API 金鑰（例如 OPENAI_API_KEY）。
  • 設定預設 Agent 模型與溝通語言（中文 / 英文）。
  • 保留 Gateway 與 Skills 區塊供後續配置。

Telegram 接入流程總覽

  1. 在 Telegram 與 BotFather 建立新的 Bot，取得 Token。
  2. 在 OpenClaw Gateway 設定中新增 Telegram Connector，填入 Bot Token。
  3. 重新啟動 Gateway，使新設定生效。
  4. 在 Telegram 搜尋你的 Bot 名稱，點選 /start 開始對話。
  5. 測試基本對話與簡單指令，確認 Agent 正常回應。
完成此步驟 = 入門 50% 完成

常見錯誤與排查

問題 可能原因 排查方向
Bot 沒有回應 Gateway 未啟動 / Token 填錯 / Telegram Webhook 設錯。 確認服務正在執行；重新產生並填入 Token。
回應延遲很久 LLM 回應時間過長或 API 設定錯誤。 檢查 LLM 日誌；改用較快模型測試。

高階開發 Week 7–10+ · 自訂 Skills / Memory / 工作流
目標：理解 Skills 與 SKILL.md 架構，能自行撰寫自訂 Skills、調整 metadata.openclaw，並規劃定時 / 條件觸發的自動化工作流。

1. 檔案與目錄結構

  • 每個 Skill 一個獨立資料夾（含程式碼與 SKILL.md）。
  • SKILL.md：自然語言描述技能行為、使用方法、邊界與安全限制。
  • metadata.openclaw（YAML）：定義 emoji、依賴、安裝命令等。

2. 設計原則

  • 「對 Agent 說話」：用自然語言描述，而不是只寫技術細節。
  • 清楚列出「可以做什麼 / 不可以做什麼」。
  • 每個 Skill 單一職責，避免過度肥大。

SKILL.md 標準結構範例（適合貼到文件或 Google Sites 中）：

# Skill 名稱：Daily Market Briefing

## 這個 Skill 在做什麼
- 每個交易日早上，整理全球股市、商品、匯率與重大新聞，輸出一份 Markdown 晨報。

## 什麼時候應該使用這個 Skill
- 使用者提到「晨報」、「早上看盤」、「今天市場怎麼樣」時。
- 使用者是交易員 / 風控 / 投資相關角色。

## 執行步驟
1. 透過市場資料 API 抓取最新指數 / 期貨 / 匯率。
2. 透過新聞 API 抓取與金融相關的重要標題與簡短內容。
3. 將結果整理成以下區塊：
   - Overnight Summary
   - Index & Futures
   - FX & Commodities
   - Today Watchlist & Event
4. 將整理好的報告以 Markdown 格式回傳或寫入指定筆記系統。

## 不應該做的事
- 不要主動下單或調整實際倉位。
- 不要在沒有明確授權時寫入高風險位置（例如：風控系統真實參數）。

晨間市場簡報工作流

適合交易員 / 量化研究人員：

  • 每天 08:30 自動觸發。
  • 呼叫「市場資料 + 新聞 + 技術指標」相關 Skills。
  • 整理成 Markdown 報告後，寫入 Obsidian / Notion，並推送到 Telegram 群組。

開發團隊每日 Digest

適合工程團隊：

  • 每天收集 GitHub PR / Issue / CI 結果。
  • 用 summarize 生成短版與長版摘要。
  • 透過 Slack / Email 發送給團隊成員。
